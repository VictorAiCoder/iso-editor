# Математические принципы изометрического пространства для игрового движка

## 1. Основы изометрической проекции

### Классическая изометрия
Истинная изометрия — проекция, при которой все три оси координат образуют углы 120° между собой, а углы наклона осей к плоскости экрана составляют 30°.

**Матрица истинной изометрической проекции:**
```
| cos(30°)   cos(150°)   0 |     | √3/2   -√3/2   0 |
| sin(30°)   sin(150°)  -1 |  =  | 1/2     1/2   -1 |
```

### Игровая (диметрическая) изометрия 2:1
В играх обычно используют соотношение 2:1 (угол ≈ 26.57°, tan = 0.5) — это удобно для пиксельной графики.

**Формулы преобразования (world → screen):**
```
screenX = (worldX - worldY) * (tileWidth / 2)
screenY = (worldX + worldY) * (tileHeight / 2)
```

**Обратное преобразование (screen → world):**
```
worldX = (screenX / (tileWidth/2) + screenY / (tileHeight/2)) / 2
worldY = (screenY / (tileHeight/2) - screenX / (tileWidth/2)) / 2
```

## 2. Матричное представление

**Прямая матрица проекции (2:1):**
```
| TW/2   -TW/2 |   | worldX |   | screenX |
| TH/2    TH/2 | * | worldY | = | screenY |
```

**Обратная матрица:**
```
| 1/TW    1/TH |
|-1/TW    1/TH |
```

Детерминант: `TW * TH / 2` — никогда не равен нулю, значит трансформация всегда обратима.

## 3. Учёт высоты (3D в изометрии)

Для тайлов с высотой Z:
```
screenX = (worldX - worldY) * (tileWidth / 2)
screenY = (worldX + worldY) * (tileHeight / 2) - worldZ * tileHeightZ
```

## 4. Сетка и определение тайла

### Определение тайла по координатам мыши
```
tileX = floor(worldX)
tileY = floor(worldY)
```

### Алгоритм "точка в ромбе" (для точного клика)
Ромб описывается уравнением:
```
|x - cx| / (w/2) + |y - cy| / (h/2) ≤ 1
```

## 5. Порядок отрисовки (Depth Sorting)

### Простая сортировка (painter's algorithm)
Порядок прорисовки определяется суммой:
```
depth = worldX + worldY + worldZ
```
Чем больше depth — тем позже рисуется (ближе к зрителю).

### Топологическая сортировка для сложных объектов
Для объектов размера > 1 тайла используется граф зависимостей:
- Объект A рисуется перед B, если AABB(A) "позади" AABB(B)
- Условие "позади": `A.maxX ≤ B.minX OR A.maxY ≤ B.minY OR A.maxZ ≤ B.minZ`

## 6. Базисные векторы изометрии

```
ê_x = ( TW/2,  TH/2)   // направление оси X в экране
ê_y = (-TW/2,  TH/2)   // направление оси Y в экране
ê_z = (    0,  -TZ )   // направление оси Z (вверх)
```

Любая позиция: `P_screen = worldX·ê_x + worldY·ê_y + worldZ·ê_z`

## 7. Камера и скроллинг

```
screenX = (worldX - worldY) * (TW/2) - cameraX
screenY = (worldX + worldY) * (TH/2) - cameraY
```

### Зум
```
screenX = ((worldX - worldY) * (TW/2) - cameraX) * zoom
screenY = ((worldX + worldY) * (TH/2) - cameraY) * zoom
```

## 8. Поиск пути (Pathfinding)

### Расстояния на изометрической сетке
- **Манхэттенское** (4 направления): `|dx| + |dy|`
- **Чебышёва** (8 направлений): `max(|dx|, |dy|)`
- **Октильное** (диагонали стоят √2): `max(|dx|,|dy|) + (√2-1)·min(|dx|,|dy|)`

Эвристика для A* на изометрии — октильная даёт оптимальный результат.

## 9. Frustum Culling (отсечение)

Видны только тайлы, экранные координаты которых попадают в:
```
screenX ∈ [-TW, viewportWidth + TW]
screenY ∈ [-TH, viewportHeight + TH]
```

Диапазон мировых координат для отрисовки:
```
worldX_min = (cameraX/(TW/2) + cameraY/(TH/2)) / 2 - margin
worldX_max = ((cameraX+W)/(TW/2) + (cameraY+H)/(TH/2)) / 2 + margin
```

## 10. Преобразование направлений (повороты камеры)

Для вращения мира на 90°:
```
(x, y) → (y, -x)    // поворот на 90° по часовой
(x, y) → (-y, x)    // против часовой
(x, y) → (-x, -y)   // 180°
```

## 11. Коллизии в изометрии

Логически коллизии считаются в **мировых координатах** (декартова система), а не в экранных. Это упрощает физику:
- AABB-коллизии: стандартные
- Окружности: стандартные
- Преобразование только для отрисовки

## 12. Освещение и нормали

Псевдо-3D эффект освещения:
```
brightness = N · L
```
Где N — нормаль грани (в мировой системе), L — направление света.

Для тайлов с разной высотой соседей нормаль грани:
```
N = normalize(cross(v1-v0, v2-v0))
```

## 13. Триггеры на основе луча (Ray-casting)

Луч в мировых координатах:
```
P(t) = origin + t·direction
```

Пересечение с горизонтальным тайлом высоты h:
```
t = (h - origin.z) / direction.z
hitX = origin.x + t·direction.x
hitY = origin.y + t·direction.y
```

## 14. Оптимизации

- **Чанки**: разделение карты на блоки N×N для batch-rendering
- **Pre-rendering**: статичные участки рендерятся в текстуру
- **Spatial hashing**: для быстрого поиска объектов по позиции
- **Z-pre-pass sorting**: сортировка через bucket sort по depth

## 15. Ключевые константы для движка

```
TILE_WIDTH  = 64    // ширина тайла в пикселях
TILE_HEIGHT = 32    // высота (TILE_WIDTH / 2)
TILE_Z      = 16    // высота "этажа"
HALF_W = TILE_WIDTH / 2
HALF_H = TILE_HEIGHT / 2
```

Эти принципы образуют математическую основу любого изометрического движка — от простых стратегий до сложных RPG с многоэтажными зданиями.
